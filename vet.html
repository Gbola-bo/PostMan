Run this before adding any new .psd to the app. It checks the file against the template spec and, if it passes, gives you the lightweight index for templates/manifest.json plus a separate detail file per template - keeping the dashboard fast to load no matter how many templates you add, since it never has to download the heavy per-layer data just to show a card.

      Detail files

      One per template - this is the heavy per-layer data, loaded only when someone actually opens that specific template, never upfront. Save each one into templates/details/ using the filename shown.

      Also copy the actual .psd file into the templates/ folder (matching the index's "file" field), and every downloaded preview image (the dashboard one, plus all the per-slide ones) into templates/thumbnails/ - each named to match its path in the index - then commit and push everything, including the new templates/details/*.json files.
